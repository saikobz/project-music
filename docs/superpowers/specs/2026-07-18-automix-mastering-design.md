# HarmoniQ: Auto Mix & Mastering Features Design

## 1. ภาพรวม (Overview)
เอกสารนี้กำหนดโครงสร้างและแนวทางการออกแบบสำหรับการเพิ่มฟีเจอร์ปรับแต่งเสียงระดับมืออาชีพ 2 รายการให้กับเว็บแอปพลิเคชัน HarmoniQ:
- **Auto Vocal Polish & Ride (จัดการเสียงร้องอัตโนมัติ):** ใช้ AI ในการประมวลผลเสียงร้องเพื่อเพิ่มความคมชัดและควบคุมระดับความดังให้สม่ำเสมอ
- **Streaming Platform Standardizer (ปรับระดับความดัง LUFS):** ระบบมาสเตอริ่งอัตโนมัติเพื่อให้ความดังของเพลงได้มาตรฐานตรงตามแพลตฟอร์มเป้าหมาย

ฟีเจอร์เหล่านี้จะทำงานในรูปแบบ "Interactive Mix & Master" ซึ่งผู้ใช้จะมีส่วนร่วมและสามารถควบคุมผลลัพธ์สุดท้ายได้ ในขณะที่ระบบจัดการงานประมวลผลเสียงที่ซับซ้อนให้ทั้งหมด

## 2. การออกแบบส่วนแสดงผล (Frontend Design / UI)

### 2.1 ส่วนควบคุม Vocal Polish (Vocal Polish UI)
- **ตำแหน่งที่ตั้ง:** ในหน้าต่างหลักที่แสดงเครื่องเล่นแบบแยกแทร็ก (WaveSurfer.js) โดยเจาะจงที่แผงควบคุมของแทร็ก "Vocals"
- **องค์ประกอบ (Component):** ปุ่มเปิด/ปิด (Toggle) หรือสไลเดอร์ มีข้อความระบุว่า "✨ AI Vocal Polish"
- **การตอบสนอง (Interaction):**
  - เมื่อผู้ใช้กดเปิด หน้าเว็บจะเรียกใช้งาน API เพื่อประมวลผลแทร็กเสียงร้อง
  - UI จะแสดงสถานะกำลังโหลดที่แทร็กเสียงร้อง
  - เมื่อประมวลผลเสร็จสิ้น เครื่องเล่นจะเปลี่ยนไปใช้ไฟล์เสียงร้องที่ผ่านการปรับแต่งแล้วแทนชั่วคราว เพื่อให้ผู้ใช้สามารถกดฟังและเปรียบเทียบ (A/B Test) กับเสียงร้องต้นฉบับได้แบบเรียลไทม์

### 2.2 หน้าต่างส่งออกไฟล์ (Export / Master Modal)
- **ตำแหน่งที่ตั้ง:** แสดงผลขึ้นมาแทนที่การดาวน์โหลดไฟล์โดยตรงเมื่อผู้ใช้คลิกปุ่ม "Export Mix" หรือ "Download"
- **องค์ประกอบ (Component):** หน้าต่าง Modal สำหรับให้เลือกเป้าหมายมาตรฐาน LUFS
- **ตัวเลือกที่มีให้:**
  - `Spotify / YouTube (-14 LUFS)` (ค่าเริ่มต้น)
  - `Apple Music (-16 LUFS)`
  - `CD / Club / Loud (-9 LUFS)`
- **การตอบสนอง (Interaction):**
  - ผู้ใช้เลือกเป้าหมายและคลิกปุ่ม "Start Mastering & Download"
  - ระบบจะแสดงแถบความคืบหน้า (Progress Bar) หรือไอคอนกำลังโหลด (Spinner) ในระหว่างที่ Backend กำลังทำการผสมเสียงและมาสเตอริ่ง
  - เมื่อการประมวลผลเสร็จสิ้น ระบบจะบังคับให้ดาวน์โหลดไฟล์ `.wav` ตัวสมบูรณ์

## 3. การออกแบบระบบเบื้องหลัง (Backend Design / API & Audio Processing)

### 3.1 เครื่องมือจัดการเสียงร้อง (Vocal Polish Engine)
- **Endpoint:** `POST /api/process/vocal-polish`
- **ข้อมูลขาเข้า (Request):** รับค่า `file_id` หรือเส้นทางของไฟล์แทร็กเสียงร้องที่แยกไว้แล้ว
- **ขั้นตอนการประมวลผล (ใช้ `pedalboard` หรือ `librosa`):**
  1. **Dynamic EQ / De-esser:** ลดความถี่ที่แหลมบาดหู หรือเสียงเสียดแทรก (เช่น ช่วง 5kHz - 8kHz)
  2. **Compression:** ควบคุมความดังของเสียงร้องให้คงที่ ไม่ให้มีบางท่อนที่เบาหรือดังจนเกินไป
  3. **Air EQ Boost:** เพิ่มความถี่สูง (High-shelf) เล็กน้อย (เช่น +2dB ที่ 10kHz) เพื่อให้เสียงร้องดูใสและมีมิติ
- **ข้อมูลขาออก (Response):** ส่งกลับ URL หรือเส้นทางของไฟล์ชั่วคราว `_polished.wav` ที่ปรับแต่งเสร็จแล้ว

### 3.2 เครื่องมือปรับระดับความดัง (LUFS Mastering Engine)
- **Endpoint:** `POST /api/process/mastering`
- **ข้อมูลขาเข้า (Request):** 
  - การตั้งค่ามิกซ์ (ระดับความดังของแต่ละแทร็ก, สถานะ Mute, และเส้นทางไฟล์เสียงร้องที่ผ่าน/ไม่ผ่านการ Polish)
  - ค่า `target_lufs` เป้าหมาย (เช่น -14.0)
- **ขั้นตอนการประมวลผล:**
  1. **Mixdown:** นำทุกแทร็กเสียงมารวมกันตามการตั้งค่าความดังของผู้ใช้
  2. **Loudness Measurement:** ใช้โมดูล `pyloudnorm` ในการวัดค่า LUFS โดยรวมของแทร็กที่ถูกมิกซ์แล้ว
  3. **Gain Adjustment:** คำนวณส่วนต่างของ Gain เพื่อให้ได้ค่าตรงกับ `target_lufs` และปรับความดังของเพลง
  4. **True Peak Limiting:** นำไฟล์ที่ได้มาผ่าน True Peak Limiter (เช่น ใช้ `pedalboard.Limiter` ตั้งค่าไว้ที่ -1.0 dBTP) เพื่อป้องกันไม่ให้เสียงแตก (Clipping)
- **ข้อมูลขาออก (Response):** ส่งกลับไฟล์ `.wav` ตัวสมบูรณ์เพื่อให้ผู้ใช้ดาวน์โหลด

## 4. ความต้องการทางเทคนิคและไลบรารีที่จำเป็น (Technical Requirements & Dependencies)
- **แพ็กเกจสำหรับ Backend:** `pedalboard` (สำหรับการทำ EQ, Compression, Limiting), `pyloudnorm` (สำหรับการวัดและปรับค่า LUFS)
- **การจัดการไฟล์:** ต้องมั่นใจว่าไฟล์ชั่วคราวที่เป็นไฟล์ polished และไฟล์ master จะถูกรวมอยู่ในระบบลบไฟล์อัตโนมัติ (Cleanup tasks) ที่มีอยู่แล้ว เพื่อป้องกันปัญหาพื้นที่ดิสก์เต็ม
