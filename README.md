# HarmoniQ — AI Music Stem & Mastering Toolkit

> เอกสารนี้ออกแบบให้ **HR และผู้ที่มาดูผลงาน** เข้าใจได้ง่าย โดยไม่จำเป็นต้องมีพื้นฐานด้านเทคนิค

---

## โปรเจกต์นี้คืออะไร?

**HarmoniQ** เป็นเว็บแอปพลิเคชันสำหรับ **จัดการและปรับแต่งไฟล์เสียงเพลง** โดยใช้ **ปัญญาประดิษฐ์ (AI)** เป็นตัวช่วยหลัก

ผู้ใช้สามารถอัปโหลดไฟล์เพลง (รูปแบบ WAV) แล้วใช้เครื่องมือบนหน้าเว็บเพื่อ:

- **แยกเสียง** ออกเป็นส่วนต่าง ๆ เช่น เสียงร้อง, กลอง, เบส, เครื่องดนตรีอื่น ๆ
- **ปรับคุณภาพเสียง** ด้วย AI หรือเครื่องมือมืออาชีพ (EQ, Compressor)
- **วิเคราะห์เพลง** เช่น จังหวะ (Tempo), คีย์ (Key), โน้ตเสียงหลัก
- **ฟังและดาวน์โหลด** ผลลัพธ์กลับไปใช้ต่อได้ทันที

เปรียบเทียบง่าย ๆ: คล้ายเครื่องมือในสตูดิโอเสียงหรือโปรแกรมตัดต่อเพลง แต่ทำผ่านเว็บเบราว์เซอร์ และมี AI ช่วยลดขั้นตอนที่ซับซ้อน

---

## ทำไมถึงสร้างโปรเจกต์นี้?

การแยกเสียงและปรับมิกซ์เพลงมักต้องใช้ **ซอฟต์แวร์ราคาแพง** และ **ความรู้เฉพาะทาง** โปรเจกต์นี้แสดงให้เห็นว่าสามารถนำ AI มาช่วยให้งานเหล่านี้ **เข้าถึงได้ง่ายขึ้น** ผ่านหน้าเว็บที่ใช้งานเข้าใจไม่ยาก

---

## ฟีเจอร์หลัก (อธิบายแบบเข้าใจง่าย)

| ฟีเจอร์                        | ทำอะไรได้                                 | ตัวอย่างการใช้งาน                                         |
| ------------------------------ | ----------------------------------------- | --------------------------------------------------------- |
| **แยกเสียง (Stem Separation)** | แยกเพลง 1 ไฟล์ ออกเป็นหลายแทร็ก           | เอาเสียงร้องออกเพื่อทำคาราโอเกะ, หรือดึงเฉพาะกลองมา remix |
| **Auto-EQ (AI)**               | ให้ AI ช่วยปรับโทนเสียงให้เหมาะกับแนวเพลง | ปรับเสียงให้สมดุลโดยไม่ต้องตั้งค่า EQ เองทีละจุด          |
| **EQ & Compressor**            | เครื่องมือปรับเสียงแบบที่ช่างเสียงใช้     | ทำให้เสียงชัดขึ้น ดัง-เบาสมดุล หรือกด dynamic range       |
| **Pitch Shift**                | เปลี่ยนระดับเสียง (สูง-ต่ำ)               | ปรับคีย์เพลงให้เข้ากับเสียงร้อง                           |
| **วิเคราะห์เพลง**              | อ่านข้อมูลพื้นฐานของเพลง                  | ดู BPM, คีย์เพลง, โน้ตเสียงหลัก                           |
| **เครื่องเล่นหลายแทร็ก**       | ฟังแต่ละส่วนของเพลงแยกกัน                 | เปิด-ปิดแต่ละแทร็ก, เลื่อนจุดฟัง, แล้วดาวน์โหลด           |

## ภาพรวมการทำงาน (3 ขั้นตอน)

```
1. อัปโหลดไฟล์ WAV  ──►  2. เลือกเครื่องมือที่ต้องการ  ──►  3. ฟังผลลัพธ์ / ดาวน์โหลด
        │                           │                                    │
   หน้าเว็บ (Frontend)         AI + ระบบประมวลผลเสียง              ได้ไฟล์แยกสเตม
                              (Backend)                           หรือไฟล์ที่ปรับแล้ว
```

1. **หน้าเว็บ** — ผู้ใช้อัปโหลดไฟล์และเลือกว่าจะทำอะไร
2. **ระบบหลังบ้าน** — ประมวลผลเสียงด้วย AI และอัลกอริทึมเสียง
3. **ผลลัพธ์** — แสดงบนหน้าเว็บ ฟังได้ทันที และดาวน์โหลดกลับไปใช้ต่อ

## สำหรับผู้พัฒนาที่ต้องการรันโปรเจกต์

คลิกเพื่อดูวิธีติดตั้งและรัน (Technical Quick Start)

### ความต้องการระบบ

- Python 3.10 (64-bit)
- Node.js 20+
- FFmpeg ต้องอยู่ใน `PATH`
- GPU + CUDA (ถ้ามี) จะช่วยให้ Open-Unmix ทำงานเร็วขึ้น แต่ไม่บังคับ

### Quick Start (Windows PowerShell)

**1. รัน Backend**

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r backend/requirements.txt
uvicorn backend.main:app --reload --port 8000
```

**2. รัน Frontend** (เทอร์มินัลใหม่)

```powershell
npm install
"NEXT_PUBLIC_API_BASE=http://localhost:8000" | Set-Content .env.local
npm run dev
```

**3. เปิดใช้งาน**

- Frontend: `http://localhost:3000`
- Backend health check: `http://localhost:8000/health`

### Environment Variables

**Frontend** (`.env.local`):

```env
NEXT_PUBLIC_API_BASE=http://localhost:8000
```

**Backend** (ตั้งก่อนรัน):

| ตัวแปร                 | ค่าเริ่มต้น             | ความหมาย                               |
| ---------------------- | ----------------------- | -------------------------------------- |
| `ALLOWED_ORIGINS`      | `http://localhost:3000` | origin ที่ frontend อนุญาตให้เรียก API |
| `SEPARATE_TTL_SECONDS` | `1200`                  | เวลาก่อนลบไฟล์ชั่วคราว (วินาที)        |

### โฟลเดอร์เอาต์พุต

| โฟลเดอร์      | เก็บอะไร                     |
| ------------- | ---------------------------- |
| `uploads/`    | ไฟล์ต้นฉบับและ ZIP ดาวน์โหลด |
| `separated/`  | ไฟล์สเตมที่แยกแล้ว           |
| `eq_applied/` | ไฟล์ที่ผ่าน EQ / Auto-EQ     |
| `compressed/` | ไฟล์ที่ผ่าน compression      |

### แก้ปัญหาเบื้องต้น

- `ffmpeg` ใช้ไม่ได้ → ติดตั้ง FFmpeg และเพิ่มลง `PATH`
- Frontend เรียก Backend ไม่ได้ → ตรวจ `NEXT_PUBLIC_API_BASE` และ `ALLOWED_ORIGINS`
- เปลี่ยนพอร์ต backend → อัปเดต `NEXT_PUBLIC_API_BASE` ให้ตรงกัน
